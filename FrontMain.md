
# 📘 HTML: meta-теги и семантические теги

## 🧩 1. Что такое `<meta>` и зачем он нужен

### 📖 Определение:
`<meta>` — это тег, который **описывает метаданные документа**, то есть информацию *о странице*, но **не отображается** напрямую на сайте.

Размещается **внутри `<head>`**, и браузеры, поисковики и социальные сети используют эти данные для:
- правильного отображения страницы;
- SEO (поисковая оптимизация);
- расшаривания ссылки (preview в Telegram, Facebook и т.д.);
- управления кодировкой и viewport’ом.

---

### ⚙️ Основные типы meta-тегов:

| Назначение | Пример | Что делает |
|-------------|---------|------------|
| Кодировка | `<meta charset="UTF-8" />` | Указывает кодировку документа (UTF-8 — стандарт, поддерживает все языки). |
| Viewport (адаптив) | `<meta name="viewport" content="width=device-width, initial-scale=1.0" />` | Делает сайт адаптивным: масштабируется под ширину устройства. </br> width=device-width: Устанавливает ширину viewport равной ширине экрана устройства (в пикселях). </br>initial-scale=1.0: Задаёт начальный уровень масштабирования (1.0 означает 100% без зума).|
| Описание (SEO) | `<meta name="description" content="Онлайн-магазин электроники" />` | Краткое описание сайта (отображается в результатах поиска). |
| Ключевые слова (устарел) | `<meta name="keywords" content="frontend, react, vue" />` | Раньше использовалось для SEO, сейчас игнорируется поисковиками. |
| Автор | `<meta name="author" content="Иван Иванов" />` | Информация об авторе страницы. |
| Robots | `<meta name="robots" content="index, follow" />` | Настройки для поисковых роботов: индексировать страницу или нет. Процесс, при котором поисковые системы добавляют страницы сайта в свою базу данных (индекс), чтобы потом показывать их в результатах поиска. Не индексируется (Она закрыта от роботов (robots.txt, noindex или nofollow, На неё нет ссылок (внутренних или внешних). Ускорение индексации - Отправить XML-карту сайта (sitemap.xml)|
| Open Graph (для соцсетей) | `<meta property="og:title" content="Мой блог" />` | Используется Facebook, Telegram и др. для предпросмотра ссылки. |
| Twitter Card | `<meta name="twitter:card" content="summary_large_image" />` | Определяет стиль карточки при расшаривании ссылки в Twitter (X). |

---

### 💡 Пример полного набора `<meta>`:
```html
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <meta name="description" content="Блог о веб-разработке" />
  <meta name="author" content="Frontend Dev" />
  <meta name="robots" content="index, follow" />

  <!-- Open Graph -->
  <meta property="og:title" content="Frontend блог" />
  <meta property="og:description" content="Статьи о React, Vue, TypeScript" />
  <meta property="og:image" content="https://example.com/preview.png" />
  <meta property="og:url" content="https://example.com/" />

  <!-- Twitter -->
  <meta name="twitter:card" content="summary_large_image" />
  <meta name="twitter:title" content="Frontend блог" />
</head>
```

---

### 💬 Частые собеседовательные вопросы про `<meta>`:

1. **Зачем нужен `<meta charset="UTF-8">`?**  
   ➜ Чтобы браузер знал, как интерпретировать символы (кириллица, emoji и т.д.).  
2. **Что делает `<meta name="viewport" ...>`?**  
   ➜ Настраивает масштаб страницы для мобильных устройств.  
3. **Что делает `<meta name="robots" content="noindex, nofollow">`?**  
   ➜ Запрещает поисковым системам индексировать страницу.  
4. **Чем отличаются `name` и `property`?**  
   ➜ `name` используется для стандартных метаданных, `property` — для Open Graph.

## 🧩 2. Семантические теги HTML

### 📖 Что это такое:
**Семантика** — это смысл.  
Семантические теги помогают браузерам, поисковым системам и скринридерам *понимать структуру страницы*.

Пример:
```html
<header>Заголовок сайта</header>
<nav>Меню</nav>
<main>
  <article>
    <h1>Статья</h1>
    <section>Основной контент</section>
  </article>
  <aside>Реклама</aside>
</main>
<footer>Контакты</footer>
```

---

### 📘 Основные семантические теги:

| Тег | Назначение | Примечание |
|------|-------------|------------|
| `<header>` | Верхняя часть страницы или раздела | Может быть несколько на странице |
| `<nav>` | Основная навигация | Меню, ссылки |
| `<main>` | Основное содержимое документа | Только один на странице |
| `<section>` | Тематическая секция контента | Можно иметь заголовок `<h2>` |
| `<article>` | Независимый блок контента | Можно копировать отдельно |
| `<aside>` | Побочный контент (сайдбар, реклама) | Не часть основного контента |
| `<footer>` | Подвал сайта | Может включать контакты |
| `<figure>` и `<figcaption>` | Изображение с подписью | Для медиа |
| `<time>` | Метка времени | Можно добавить `datetime="2025-11-08"` |

---

### 💬 Частые собеседовательные вопросы про семантику:

1. **Зачем использовать `<article>` вместо `<div>`?**  
   ➜ `<article>` сообщает, что это самостоятельный кусок контента.  
2. **Можно ли иметь несколько `<header>` или `<footer>`?**  
   ➜ Да, если они принадлежат разным секциям.  
3. **Чем `<section>` отличается от `<div>`?**  
   ➜ `<section>` обозначает логический раздел, `<div>` — просто контейнер.  
4. **Что делает тег `<main>` и сколько его может быть?**  
   ➜ Определяет основное содержимое, должен быть только один.  
5. **Почему важно использовать `<nav>`?**  
   ➜ Для доступности и SEO, помогает навигации скринридеров.
